<objective>
Add boundary-only mid-run disk-full guard so a recon run aborts cleanly with `ENOSPC`-style error rather than producing zero-byte or truncated outputs when the working filesystem fills mid-scan. Closes RESIL-02.

Purpose: Today, `check_disk_space()` (modules/utils.sh:421) is only called once at the start of each recon mode (modules/modes.sh:23). If disk fills mid-run (e.g., during a 2GB gotator permutation write or a long DNS resolve), tools fail silently and downstream pipeline stages see corrupt partial outputs. This plan installs the same `df`-based check at every `start_func`/`end_func` boundary so the next function entry aborts the run before any further damage.

Output: New private helpers `_check_disk_mid_run` and `_abort_disk_full` in `modules/utils.sh`. `start_func` and `end_func` in `modules/core.sh` call them at the documented anchors (top-of-start, tail-of-end) so the check runs at boundary cadence with negligible overhead (~1ms per call × ~100 boundaries per scan).
</objective>
