<objective>
Add `PARALLEL_JOB_TIMEOUT_SECONDS` enforcement so stuck tools no longer stall a parallel batch, and set non-zero DNS timeout defaults so a hung resolver does not block a run indefinitely. Closes RESIL-03 and PERF-02.

Purpose: Today, the heartbeat loop at `lib/parallel.sh:432-465` polls alive PIDs every `PARALLEL_HEARTBEAT_SECONDS` but provides no kill mechanism for stuck jobs. A single hung tool stalls the entire batch barrier-sync. Separately, `DNS_BRUTE_TIMEOUT=0` and `DNS_RESOLVE_TIMEOUT=0` at `reconftw.cfg:387-388` ship as disabled; `_run_dns_with_heartbeat` (modules/utils.sh:1434) already supports non-zero durations but the defaults never trigger.

Output: New private helper `_timeout_kill_job` in `lib/parallel.sh`; timeout-check inserted in both heartbeat loops (batch-flush at :432-465 and final-wait at :537-569); reason rendering for FAIL badges widened in `_parallel_emit_job_output`; four config lines in `reconftw.cfg` (two value changes for DNS, two new parallel knobs). No CLI flag added (env-var/cfg override is sufficient for v1 per CONTEXT §Established Patterns).
</objective>
