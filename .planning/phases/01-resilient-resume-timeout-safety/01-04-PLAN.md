<objective>
Restore the `.inprogress_<fn>` resume indicator for SIGINT/SIGTERM interruptions (Gap A / CR-01 / SC1 partial). Closes the gap by introducing a `_RECON_CLEAN_EXIT` flag that gates `_cleanup_inprogress` — the EXIT trap sweeps sentinels only when a clean traversal explicitly opts in. SIGINT/SIGTERM does not set the flag, so the `.inprogress_<fn>` files survive `cleanup_on_exit`'s `exit 130` and the next run's WARN resume banner fires correctly. Also updates the misleading start_func comment at `modules/core.sh:1431-1434` and reconciles `01-CONTEXT.md` D-02 wording with implemented semantics.

Purpose: 01-VERIFICATION.md marked SC1 PARTIAL because `cleanup_on_exit` (INT/TERM handler) ends in `exit "$exit_code"` (modules/utils.sh:145). Per Bash semantics, an explicit `exit` from inside an INT/TERM trap fires the EXIT trap, which currently runs `_cleanup_inprogress` unconditionally and wipes every `.inprogress_<fn>` (verified empirically by `/tmp/sc1_v3.sh`). Result: Ctrl+C resume indicator never appears — the most common interruption path silently fails the SC1 indicator guarantee. The verifier flagged this exact gap and the 01-REVIEW.md CR-01 advisory described the clean-exit-flag fix; this plan implements it.

Output:
- `modules/modes.sh`: declare `_RECON_CLEAN_EXIT=false` at the top of `start()` (after `global_start=$(date +%s)` and before `set +m`).
- `modules/modes.sh`: at the END of `end()` (the canonical workflow wrap-up), set `_RECON_CLEAN_EXIT=true` as the last executable statement so any successful traversal flips the flag before the shell exits.
- `modules/utils.sh`: gate `_cleanup_inprogress` body on `${_RECON_CLEAN_EXIT:-false} == "true"`; early-return otherwise. `cleanup_on_exit` itself is NOT modified.
- `modules/core.sh:1431-1434`: rewrite the `start_func` comment to reflect the gated-trap semantics.
- `01-CONTEXT.md` §Decisions D-02: rewrite the body sentence so the locked decision matches the implemented behavior. Decision ID and intent are preserved; only wording changes.

No new tests; Phase 4 owns coverage (`/gsd-verifier` will not re-flag SC1 once smoke 1 passes).
</objective>
