<objective>
Add `.inprogress_<fn>` sentinel lifecycle so interrupted runs surface a clear resume indicator and only re-execute the function that was killed mid-run. Closes RESIL-01.

Purpose: Today, `end_func` writes `.<fn>` only on clean completion (`modules/core.sh:1465`). An interrupted function (Ctrl-C, OOM, crash) leaves NO marker, so a `PRESERVE=true` re-run cannot distinguish "function never ran" from "function crashed mid-run". This plan adds a touch-on-start / remove-on-end sentinel plus trap-based cleanup so leftover sentinels on the next invocation become the resume signal.

Output: `start_func`/`end_func` write and clear `.inprogress_<fn>`; `start()` in `modules/modes.sh` installs a separate `EXIT`-only trap that calls a new lightweight `_cleanup_inprogress` helper (it clears stale sentinels on graceful exit without any banner, kill, or forced exit code). The existing `cleanup_on_exit` (modules/utils.sh:116-146) stays bound to `INT/TERM` only and is NOT modified. `FORCE_RESCAN=true` adds an explicit `.inprogress_*` orphan clear and start() emits a one-line resume WARN banner when leftover sentinels are detected from a prior crash.
</objective>
