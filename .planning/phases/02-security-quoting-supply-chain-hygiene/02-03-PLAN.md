<objective>
Three independent supply-chain hardening changes bundled in one plan:

1. **AXIOM_EXTRA_ARGS array refactor (SEC-03)** — Replace 38 inconsistent `$AXIOM_EXTRA_ARGS` / `"$AXIOM_EXTRA_ARGS"` call sites across `modules/subdomains.sh` (21 sites) and `modules/web.sh` (15 sites + 1 local-pattern block) with a single global `AXIOM_EXTRA_ARGS_ARR=()` array parsed once in `reconftw.sh`. Eliminate the per-function `axiom_extra_args=()` local pattern in `webprobe_full` (D-09).
2. **Installer SHA256 verification (SEC-04 part 1)** — Wire env-var-driven `verify_sha256` checks into `install_rust_uv()` behind `RUSTUP_INSTALLER_SHA256` and `UV_INSTALLER_SHA256`. Backwards-compatible: unset env vars preserve pre-change behaviour.
3. **`tools.lock` manifest (SEC-04 part 2)** — Pin exactly 5 stability-critical Go tools (`nuclei`, `httpx`, `ffuf`, `puredns`, `subfinder`) in a new repo-root `tools.lock` file. `install.sh` reads the manifest before the Go install loop; pinned tools install at the recorded version, unpinned tools fall through to `@latest`.

Purpose: SEC-03 closes a "word-splitting an unquoted variable" anti-pattern that breaks when `AXIOM_EXTRA_ARGS="--rate-limit 100 --threads 50"` is set as intended (the `"$AXIOM_EXTRA_ARGS"` quoted form makes the whole string a single token; the bare form works but is indistinguishable from accidental unquoting). SEC-04 addresses the audit finding that the installer downloads and executes upstream scripts without integrity verification AND that commit `71653984` ("remove pinned versions") removed all Go-tool version pins, leaving every install at the mercy of `@latest`. Env-var-driven pinning + a 5-tool lock manifest matches the existing `GETJSWORDS_SHA256`/`AXIOM_CONFIG_SHA256` precedent and keeps maintenance surface small (D-13).

Output: One global parse in `reconftw.sh` + 36 mechanical token substitutions across 2 modules + 1 local-block deletion in `webprobe_full`; `install_rust_uv` extended with verification gates; `install_tools` extended to consult `tools.lock` before the Go install loop; one new file `tools.lock` with 5 pinned entries.

From install.sh:82-102 (verify_sha256 helper — REUSE, do not modify):
```bash
verify_sha256() {
    local file="$1"
    local expected="$2"
    local actual=""
    [[ -n "$expected" ]] || return 0   # nothing to verify
    [[ -s "$file" ]] || return 1
    if command -v sha256sum >/dev/null 2>&1; then
        actual=$(sha256sum "$file" | awk '{print $1}')
    elif command -v shasum >/dev/null 2>&1; then
        actual=$(shasum -a 256 "$file" | awk '{print $1}')
    else
        return 0  # no hashing tool available; skip (don't block install)
    fi
    [[ -n "$actual" && "$actual" == "$expected" ]]
}
```

        header "Installing Golang tools (${#gotools[@]})"

                _key="${_key#"${_key%%[![:space:]]*}"}"
